# Project Browser TUI Design

## Goal

将无参数 `td` 的 TUI 改为双栏项目浏览器：左侧显示全部项目，选择项目后右侧即时展示其未完成事项；每个事项以 Unicode box 边框卡片呈现。

## Layout

```text
┌ Projects ─────────────┐ ┌ 项目事项 ───────────────────────────────────┐
│ › Home                │ │ ┌─────────────────────────────────────────┐ │
│   Work                │ │ │ Buy milk                                │ │
│   Personal            │ │ │ #Errands · 0/1 checklist                │ │
│                        │ │ └─────────────────────────────────────────┘ │
└───────────────────────┘ └─────────────────────────────────────────────┘
↑↓ select project · Tab focus tasks · Enter detail · c/x action · r refresh · q quit
```

左栏固定占终端可用宽度的约三分之一（最小 22 列）；右栏占剩余宽度。项目为空时显示“没有项目”。项目下无未完成事项时，右栏显示“该项目没有未完成事项”。

## Interaction

- 初始焦点在项目栏，默认选择第一个项目。
- `↑` / `↓` 在当前焦点栏移动选择；项目选择变化时调用 `service.listOpenTasks(project.id)` 刷新右栏。
- `Tab` 在项目栏和事项栏之间切换焦点。
- 事项栏获得焦点后，`c` / `x` 对选中事项显示原有确认界面；确认后刷新当前项目事项。
- `r` 重新读取项目列表，并保持仍存在的项目 UUID 作为选择；不存在则选择第一项。
- `q` 退出；无 TTY 时保持现有帮助输出行为。

## Item Card

每个事项卡片使用 `┌─┐│└─┘` 绘制边框：

- 第一行显示事项标题；超出栏宽时截断并加省略号。
- 第二行显示项目内标题（若有）、标签与 checklist 进度。
- 有备注时显示首行摘要，最多一行。
- 当前事项卡片使用 cyan 边框/前景色；其他卡片保持默认终端颜色。

卡片仅是呈现层：不改变 SQLite 查询、领域 DTO 或 URL 写入逻辑。

## Testing

- 渲染测试：项目列表出现、选择项目后调用 `listOpenTasks(project.id)` 并展示对应事项。
- 键盘测试：`↓` 切换项目、`Tab` 切换焦点、`c` 仅在事项焦点且有事项时进入确认态。
- 快照/字符串测试：确认卡片包含 `┌`、`│`、`└` 和事项标题。
- 回归：现有确认测试和非 TTY 命令测试必须保持通过。

## Scope

本次不增加搜索、项目创建、事项详情页、滚动分页、鼠标操作或新的 SQLite 字段。
