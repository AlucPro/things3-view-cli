# Things 3 CLI 本地安装、配置与功能测试

## 1. 前置条件

- macOS 13 或更高版本。
- 已安装并打开过 Things 3；CLI 读取本机 Things 数据库。
- Node.js 20 或更高版本：`node --version`。
- npm：`npm --version`。

本项目**只读** Things 的 SQLite 数据库。新建、完成和取消事项会调用 Things 官方 URL Scheme，不会直接写入 `main.sqlite`。

## 2. 本地开发安装

在项目根目录执行：

```bash
npm install
npm run build
npm test
```

`npm test` 会先编译 TypeScript，再运行单元测试。当前测试不读取真实 Things 数据库，使用合成 SQLite fixture。

## 3. 配置变量

复制模板：

```bash
cp .env.example .env
```

`.env` 示例：

```dotenv
# Things 主数据库的绝对路径。路径含空格时不需要额外转义。
THINGS_DB_PATH="/Users/your-name/Library/Group Containers/JLMPQHK86H.com.culturedcode.ThingsMac/Things Database.thingsdatabase/main.sqlite"

# 只有修改既有事项（complete / cancel）需要。
# 在 Things → Settings → General → Enable Things URLs → Manage 中取得。
THINGS_AUTH_TOKEN="paste-your-token-here"

# true：禁止 add、complete、cancel；首次使用建议保持 true。
# false：允许发送官方 Things URL 写入命令。
TD_READ_ONLY=true
```

变量说明：

| 变量 | 是否必填 | 如何填写 |
| --- | --- | --- |
| `THINGS_DB_PATH` | 建议填写 | 本机 `main.sqlite` 的绝对路径；数据库目录同时可能有 `-wal` / `-shm` 文件，不能删除它们。 |
| `THINGS_AUTH_TOKEN` | `complete` / `cancel` 必填 | 从 Things 设置取得；不要提交、截图、打印或粘贴到日志。 |
| `TD_READ_ONLY` | 否 | `true` 禁止写入，`false` 允许写入。未填写等同于 `false`，因此建议显式设置。 |

`.env` 已被 Git 忽略。全局安装后的 `td` 会从**当前终端目录**加载 `.env`；也可以只在单次命令中设置环境变量：

```bash
THINGS_DB_PATH="/absolute/path/to/main.sqlite" td projects
```

## 4. 安装到终端

### 方式 A：从当前项目全局安装（推荐用于本地开发）

```bash
npm install -g .
td --version
td --help
```

修改源码后重新执行 `npm run build && npm install -g .`。

### 方式 B：用 npm link（开发时可自动引用当前项目）

```bash
npm install
npm run build
npm link
td --version
```

取消链接：

```bash
npm unlink -g things3-view-cli
```

如果提示 `td: command not found`，检查 npm 全局 bin 是否在 `PATH`：

```bash
npm prefix -g
echo "$PATH"
```

## 5. 只读功能测试

先保持 `TD_READ_ONLY=true`，确认不会发生写入。

```bash
# 帮助与版本
td --help
td --version

# 项目列表
td projects
td projects --json

# 所有未完成事项
td list
td list --json

# 按项目 UUID 或项目名称过滤
td list --project "<project-id-or-name>"
```

兼容短参数：

```bash
td -p                  # 等价于 td projects
td -l                  # 等价于 td list
td -p "<project>" -l  # 等价于 td list --project "<project>"
```

项目名称重复时，请使用完整项目 UUID；CLI 不会擅自选择其中一个项目。

## 6. 写入功能测试

写入会影响真实 Things 数据。建议先新建一个专门的测试项目，并确认 `.env` 中为：

```dotenv
TD_READ_ONLY=false
THINGS_AUTH_TOKEN="your-token"
```

### 新增事项

```bash
td add "CLI 测试事项" --project "<project-id>" --notes "由 td 本地测试创建" --tag Test
```

### 完成或取消事项

先从 `td list --json` 获取事项的**完整** `id`。完成和取消必须带 `--yes`：

```bash
td complete "<task-uuid>" --yes
td cancel "<task-uuid>" --yes
```

完成后 CLI 会重新查询数据库进行短暂核验；如果 Things 尚未刷新，可能显示“已交给 Things，但未能核验”。这表示命令已交给 Things，不表示数据库被 CLI 直接修改。

## 7. 交互模式

在真实 TTY 中不带参数执行：

```bash
td
```

快捷键：

| 按键 | 操作 |
| --- | --- |
| `↑` / `↓` | 选择事项 |
| `c` | 请求完成，随后按 `y` 确认 |
| `x` | 请求取消，随后按 `y` 确认 |
| `r` | 刷新事项列表 |
| `n` / `Esc` | 取消确认对话框 |

## 9. 常见问题

### 找不到数据库

确认路径存在：

```bash
ls -l "$THINGS_DB_PATH"
```

然后确认 Things 3 已安装并至少启动过一次。不要将数据库复制到项目中或直接修改它。

### 完成/取消提示缺少 token

在 Things 的设置中启用 Things URLs，取得 token 后设置 `THINGS_AUTH_TOKEN`，并重新打开终端或重新执行命令。

### 提示只读模式

这是预期保护。完成测试后，将 `.env` 中的 `TD_READ_ONLY` 改为 `false`；完成后建议改回 `true`。
